Prompt 41: musekit-cms — CSS Variable Migration (ALL Sections, Blog, Changelog, Legal, Marketing, SEO)
Module Repl
musekit-cms (GitHub: SpeckledDarth/musekit-cms)

Context
The CMS package is the largest visual surface in MuseKit. It renders the public-facing landing page (13 section types via LandingPageBuilder), blog posts, changelog entries, custom pages, legal pages, and marketing widgets (cookie consent, announcements, feedback, waitlist). It also handles SEO metadata generation.

Currently, the Tailwind config uses var(--) instead of hsl(var(--)) (breaking opacity modifiers), landing sections use hardcoded rounded-xl/2xl/3xl, blog/changelog components use hardcoded grays, and marketing widgets use hardcoded colors. The SEO metadata is missing a themeColor property.

After Prompt 38 (ThemeInjector) and Prompt 39 (shared Tailwind preset), all CSS variables and mappings are available.

Reference: See docs/current/css-variable-contract.md for the complete list of available CSS variables, their types, defaults, and recommended Tailwind class replacements.

The Fix
1. Update tailwind.config.ts
Import the shared preset and use it:

import { tailwindPreset } from "@musekit/design-system";
const config = {
  presets: [tailwindPreset],
  content: ["./src/**/*.{js,ts,jsx,tsx}"],
  theme: {
    extend: {
      // KEEP existing animation/keyframe configs (accordion, marquee, count-up)
      // REMOVE duplicate color definitions — they come from the preset now
    },
  },
  // ... keep existing plugins
};
This fixes the var(--) → hsl(var(--)) issue automatically.

2. Migrate ALL 13 Landing Page Sections
For EVERY file in src/landing/:

Border radius: Replace hardcoded values with semantic:

rounded-xl → rounded-lg (uses var(--radius))
rounded-2xl → rounded-lg or rounded-[var(--radius)]
rounded-3xl → rounded-[var(--radius)]
Colors: Apply standard replacement rules:

bg-white → bg-background
bg-gray-* → bg-muted or bg-accent
text-gray-* → text-foreground or text-muted-foreground
border-gray-* → border-border
Specific sections to check:
